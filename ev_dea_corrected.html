กรณีศึกษา: คัดเลือกรถยนต์ไฟฟ้าด้วย DEA Cross-Efficiency
แนวทาง: ประยุกต์ใช้วิธีของ Wichapa et al. (2021) ซึ่งรวม benevolent (RCB) และ aggressive (RCA) models เข้าด้วยกัน เพื่อให้การจัดอันดับเป็นกลางและน่าเชื่อถือ

DMU = รถยนต์ไฟฟ้าแต่ละรุ่น (8 รุ่น) | Inputs (x) = ทรัพยากรที่ต้องใช้ | Outputs (y) = ประสิทธิผลที่ได้
Inputs — ยิ่งน้อยยิ่งดี
x₁ราคาซื้อ (ล้านบาท)
x₂พลังงานชาร์จเต็ม (kWh)
x₃เวลาชาร์จ AC เต็ม (ชม.)
Outputs — ยิ่งมากยิ่งดี
y₁ระยะทางต่อชาร์จ (กม.)
y₂กำลังมอเตอร์ (kW)
y₃คะแนน NCAP (จาก 5)
ขั้นตอนการคำนวณ (Wichapa et al. 2021)
Step 1: ป้อน θ*_RCB และ θ*_RCA → สร้าง CEIM: [θ_min_ij, θ_max_ij]
Step 2: RC_ij = d⁻_ij/(d⁺_ij+d⁻_ij) โดย d⁺=(1-θ_min)+(1-θ_max), d⁻=θ_min+θ_max (Eq.12–14)
Step 3: σ_j = std(column j of CEM) → ω_j = (Σσ - σ_j) / [(n-1)·Σσ] (Eq.15)
Step 4: w_i = Σ_j ω_j · RC_ij (Eq.16) → จัดอันดับ w_i มากสุด = ดีที่สุด
หมายเหตุสำคัญ: ในทางปฏิบัติ θ*_RCB และ θ*_RCA ต้องได้จาก LP Solver (LINGO/Python) สำหรับกรณีศึกษานี้ได้สร้างค่าจากสมการ RC model (Eq.6) โดยประมาณ โดยใช้ค่า ideal/anti-ideal DMU จาก Definition 1–2 ของเปเปอร์
ข้อมูล EV — แก้ไขได้
รุ่น EV x₁ราคา (ลบ.) x₂kWh x₃ชาร์จ (ชม.) y₁กม. y₂kW y₃NCAP
RCB cross-efficiency matrix — ป้อนจาก LP solver (หรือใช้ค่าประมาณอัตโนมัติ)
θ*_RCB_ij = ค่า cross-efficiency ของ DMU_i เมื่อใช้น้ำหนักที่ได้จาก benevolent model ของ DMU_j
แนวเส้นทแยงมุม (i=j) = self-efficiency จาก CCR model (≤1) | ค่าอื่น ∈ [0,1]
DMU_i \ target_j
RCA cross-efficiency matrix — ป้อนจาก LP solver (หรือใช้ค่าประมาณอัตโนมัติ)
θ*_RCA_ij = ค่า cross-efficiency ของ DMU_i เมื่อใช้น้ำหนักที่ได้จาก aggressive model ของ DMU_j
DMU_i \ target_j
Step 1 — Cross-Efficiency Intervals Matrix (CEIM) (Eq. 11)
θ_ij^min = min(θ*_RCB_ij, θ*_RCA_ij) | θ_ij^max = max(θ*_RCB_ij, θ*_RCA_ij)
Step 2 — Combined CEM via RC index (Eq. 12–14)
d⁺_ij = (1 - θ_min_ij) + (1 - θ_max_ij) [ระยะห่างจาก PIS=1]
d⁻_ij = θ_min_ij + θ_max_ij [ระยะห่างจาก NIS=0]
RC_ij = d⁻_ij / (d⁺_ij + d⁻_ij) [relative closeness ∈ (0,1)]
Step 3 — MSDT Criteria Weights ω_j (Eq. 15)
σ_j = std deviation ของ column j ใน combined CEM (population std: หารด้วย n)
ω_j = (Σσ_j − σ_j) / [(n−1) · Σσ_j] ← σ_j สูง = criterion มีความแปรปรวนมาก → ได้ weight ต่ำ
Step 4 — Final Ranking (Eq. 16)
w_i = Σ_j (ω_j × RC_ij) → อันดับที่สูงกว่า = w_i มากกว่า
